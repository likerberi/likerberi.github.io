프로젝트 핵심 목표

30년 이상 축적된 한국 미술계 아카이브를 안전하게 계승하면서, 미래 성장을 위한 기술 기반을 새롭게 구축합니다.

레거시 청산 SEO 극대화 API 기반 분리

3대 핵심 목표

01
🔧

레거시 청산 및 현대화

기존 PHP 기반의 낡은 구조를 전면 개편하여 모바일 반응형을 지원하고 향후 확장이 용이한 구조로 재구축합니다.

  • PHP 모놀리식 → 현대적 분리 아키텍처
  • 반응형 웹 디자인 전면 적용
  • 프레임워크 기반 개발로 유지보수 용이화
  • Docker 기반 개발 환경 표준화
02
🔍

검색 엔진 최적화(SEO)

구글·네이버에서 전시 및 작가 검색 시 최상단 노출을 위한 클린 URL 체계와 구조화된 메타데이터를 전면 도입합니다.

  • 클린 URL (Clean URL) 체계 전면 도입
  • 구조화 데이터 (JSON-LD) 마크업
  • SSR/SSG 기반 빠른 초기 로딩
  • Open Graph / Twitter Card 지원
03
🔗

데이터와 화면의 분리

향후 앱(App) 개발 및 외부 API 연동을 고려해 프론트엔드와 백엔드를 분리하는 API-First 아키텍처를 적용합니다.

  • RESTful API / 향후 GraphQL 확장 가능
  • 모바일 앱 출시 시 동일 API 재활용
  • 외부 파트너 API 연동 용이
  • 프론트/백 독립 배포 가능

현황 vs 목표 비교

항목 AS-IS (현재) TO-BE (목표)
프레임워크 레거시 PHP (절차형) → Next.js 14 + Laravel 11
URL 구조 /bbs/board.php?code=art&no=1234 → /exhibitions/2024/artist-name
이미지 저장 서버 로컬 디스크 → S3 오브젝트 스토리지 + CDN
데이터베이스 단독 서버 MySQL (수동 백업) → Managed DB (자동 백업·복구)
모바일 대응 별도 모바일 페이지 (m.) → 반응형 단일 사이트
API 제공 없음 → RESTful API (앱·외부 연동)
검색 Like 쿼리 DB 풀스캔 → Elasticsearch 전문 검색

❌ 기존 (검색 불리)

/news/list.php?section=arts&page=5

💡 SEO 기대 효과

클린 URL 도입 및 SSR(서버 사이드 렌더링) 적용 시, 구글 크롤러가 콘텐츠를 완전히 인식하여 전시·작가·기사 검색어 기준 상위 노출 가능성이 크게 향상됩니다. 특히 작가명 + 전시명 조합 검색에서 직접 랜딩 페이지로 연결되는 구조를 목표로 합니다.
