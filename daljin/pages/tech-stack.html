핵심 기술 상세

⚡
Next.js 14
프론트엔드 프레임워크 · 필수
App Router를 사용한 서버 컴포넌트 기반 렌더링. SSR / SSG / ISR 혼용으로 SEO와 성능을 동시에 달성합니다. 전시·작가 상세 페이지는 정적 생성 후 CDN 배포, 실시간 검색·댓글은 클라이언트 렌더링으로 처리합니다.
선택 이유: SEO 필수 요건인 서버 사이드 렌더링 + React 생태계 + Vercel/NCP 배포 최적화
🔴
Laravel 11
백엔드 API 프레임워크 · 필수
PHP 생태계에서 가장 성숙한 MVC 프레임워크. 기존 PHP 기반 데이터를 안정적으로 이관하고 Eloquent ORM으로 복잡한 관계망(작가-전시-기사)을 표현합니다. Sanctum 기반 API 토큰 인증을 제공합니다.
선택 이유: 기존 PHP 개발자 이관 용이 + 성숙한 ORM + API 리소스 변환 기능 내장
🗄️
Managed MySQL
데이터베이스 · 필수 (네이버 클라우드)
네이버 클라우드 등 국내 퍼블릭 클라우드의 Managed DB 서비스를 필수 사용. 자동 백업(일 1회 이상), Point-in-time Recovery, 장애 자동 Failover가 지원되어 수십 년 축적 데이터의 안전한 보존을 보장합니다.
선택 이유: 자동 백업·복구 지원 필수 조건 충족 + 국내 데이터 센터 레이턴시 최소화
☁️
S3 + Cloudflare CDN
이미지 스토리지 + 전송망 · 필수
모든 이미지·파일은 반드시 오브젝트 스토리지(S3 호환)에 독립 저장하며, 서버 디스크에 직접 저장하는 것을 절대 금지합니다. Cloudflare CDN을 통해 전 세계 엣지에서 빠른 이미지 서비스를 제공합니다.
선택 이유: 서버 교체/장애 시에도 이미지 유실 방지 + 무한 확장 가능한 스토리지
🔍
선택 · 권장
Elasticsearch
전문 검색 엔진 · 선택권장
30년 치 기사·전시·작가·박물관 데이터를 DB Like 쿼리로 검색하면 성능 한계가 있습니다. Elasticsearch 도입 시 한국어 형태소 분석(nori), 자동완성, 연관 검색어, 패싯(facet) 필터 등 고급 검색 기능을 제공할 수 있습니다.
권장 이유: 대용량 아카이브 검색 특화 + 향후 추천·연관 콘텐츠 기능 확장 가능
🐳
Docker + GitHub Actions
개발 환경 및 CI/CD · 필수
Docker Compose로 로컬 개발 환경을 표준화하여 팀원 간 환경 차이를 없앱니다. GitHub Actions를 통해 코드 병합 시 자동 테스트·빌드·배포 파이프라인을 구성, 안정적인 릴리즈를 보장합니다.
선택 이유: 환경 일관성 보장 + 무중단 배포 (Blue-Green) 가능
📋 에이전시 제안서 필수 포함 사항
  • 각 기술 스택 선택에 대한 구체적 근거 및 대안 비교
  • Elasticsearch 도입 여부 및 도입 시 구체적 구성 방안
  • 클라우드 인프라 비용 예산안 (월간 운영비 포함)
